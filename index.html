Skip the download step. Build the .idbx in memory and hand it to the operating system's share sheet — AirDrop, Nearby Share, Bluetooth, LocalSend, Messages, email, whatever you have installed. Mobile only, mostly.

Requires HTTPS, a real user tap, and a browser that supports navigator.canShare({files}). Falls back to download on unsupported browsers.

How the file is structured & known limits

Format. A single binary file: 4-byte header length, then a JSON header (DB name, version, store schemas, index definitions), then concatenated record blobs. Records are serialized with structuredClone-friendly encoding so Blobs, ArrayBuffers, Dates, Maps, and Sets survive the round trip.

Same origin required. IndexedDB is partitioned per origin. Both devices must open this page from the same URL (same scheme + host + port) for the import to land in the right sandbox.

Not preserved: open cursors, ongoing transactions, encryption keys tied to the origin (e.g. WebCrypto non-extractable keys stored as references).

Tested in: Chromium 120+, Firefox 120+, Safari 17+. iOS Safari has stricter quota — large DBs may need user permission.
